Preface

All of the data models follow similar syntax for types and information:

  • A ? after the "Type" of a property means that it is a nullable field. Though it will always be returned (if it is set to be returned even if it were not nullable).
  • A ? after the "Property" name means that the property may not always be returned, but when it is, it will always the type indicated.
  • Each property can have a number of superscript note indicators after it, these indicators correspond to a note(s) at the bottom of the table.

Account

Property Type Description
id[0] String The UUID for the user.
username String The account's username
discriminator Number The account's unique number that allows there to me multiple accounts with the same username.
password[1] String? The account's password after it has been salted and hashed.
salt[0][1] String? The salt that is unique to the account when they have password auth enabled.
accounts?[3] Object The third party accounts that are linked to the user account.
games[2][4] Number[] The games that the user has registered that they own.
relations[4] Relations The relationship connections for the account.
[0]: This is a read-only property, attempting to modify it will result in an HTTP 400 error.
[1]: This property is not returned from API calls
[2]: This is not expanded to contain the full object.
[3]: This is only returned from the /users/{user} endpoints when the user that is being requested is also the authenticated user.
[4]: This property is only returned in the object when the authenticated user is requesting for themselves, or they are friends with the user that they are requesting for.

Game

Property Type Description
slug[1] String A URL-safe version of the name. Example: minecraft-java-edition
name String The formatted name of the game. Example: Minecraft: Java Edition
publisher?[2] String The publisher/developer of the game.
cover String? A URL pointing to an image that represents the game. This is automatically pulled from RAWG.io during the search request.
platforms String[] The platforms that the game supports. The specificity of the platform depends on RAWG.io as the data is pulled directly from their API.
crossplays?[2] String[][] An array of arrays that indicate what platforms are capable of cross-playing with each other.
So if the game has support for 3 platforms (A, B, and C), and platforms A and B can play together, and B and C can play together, but platforms A and C cannot play together, this would lead to the following value: [ [ "A", "B" ], [ "B", "C" ] ]
[1]: This is a read-only property. Any attempted modifications will result in an HTTP 400 response.
[2]: This is currently unused.

Relations

This object represents the user's relationships within the site.

Property Type Description
friends String[] The list of the users that the user has friended.
requests String[] The list of user IDs that are asking this user to be their friend.
